### 📋 完整提示词（可直接复制）

```text
你是一名资深的全栈开发工程师与可视化架构师。请为我设计并实现一个“图转自然语言描述”的可视化连线编辑工具。要求代码结构清晰、可本地调试、支持一键打包为 Windows .exe，并优先保证离线可用。

【🎯 项目目标】
开发一个基于 Web 技术的图形编辑器，用户可通过拖拽创建自定义元件框（如电阻、与门、自定义模块），用带属性的线连接它们，并将当前图形一键导出为结构化的自然语言描述（非代码），便于阅读或二次处理。

【⚙️ 核心功能】
1. 画布编辑
   - 支持从侧边栏拖拽节点到画布，支持缩放、平移、多选、对齐
   - 支持自定义节点类型（内置：电阻、与门；预留自定义输入）
   - 支持连线：带方向箭头，支持为连线添加元数据（如：增益、符号 -1/+1、信号类型、延迟等）
   - 双击节点/连线可弹出属性面板进行编辑

2. 自然语言转换（NLG）
   - 点击“导出描述”时，将当前图数据转换为人类可读的自然语言
   - 示例输出：“电阻R1的输出端连接至与门AND_A的输入端1，连线增益为+1，信号方向为单向。与门AND_A的输出端连接至自定义模块M2，线上标记为反馈信号（-1）。”
   - 转换逻辑需基于规则/模板实现，不依赖外部 LLM API，确保离线可用

3. 数据持久化
   - 底层使用标准 JSON 存储图结构，支持导入/导出 JSON 文件
   - 提供示例数据以便首次加载

【🛠 技术栈建议】
- 前端框架：React + Vite + TypeScript
- 画布引擎：React Flow（推荐，自带节点连线、拖拽、序列化能力）或 Konva.js
- UI 组件：Ant Design / shadcn/ui + TailwindCSS
- 本地开发：Vite Dev Server（localhost:5173）
- 桌面打包：Electron（兼容性好）或 Tauri（更轻量，需 Rust 环境）
- 请优先选择纯前端方案，开发阶段通过浏览器访问，打包阶段通过桌面壳加载

【📦 数据结构规范（JSON Schema 参考）】
```json

  ],
  "meta": { "exportNL": "..." }
}
```

【🚀 开发与打包流程要求】
1. 提供完整项目初始化命令与目录结构
2. 实现最小可用版本（MVP）：含画布、2种节点、基础连线、属性编辑、NLG导出
3. 本地开发：`npm run dev` 启动 Web 服务，浏览器访问

【📤 输出格式要求】
请按以下顺序输出：
1. 项目目录树

   - 画布主组件（节点/连线渲染）
   - 自定义节点组件（电阻、与门）
   - 连线属性编辑面板
   - 图转自然语言转换逻辑（函数+模板）
   - 本地服务与打包配置
